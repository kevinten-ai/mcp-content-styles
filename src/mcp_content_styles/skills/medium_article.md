## Format Requirements

### Format Mode (mode="format")
Adjust content to Medium article format:
- Clean headings with `##` (no `#` - Medium uses title field)
- Short paragraphs (2-4 sentences)
- Pull quotes using `>` for key insights
- Minimal code (use GitHub Gists for long code)
- Add section breaks with `---`

### Rewrite Mode (mode="rewrite")
Rewrite as a Medium-quality technical essay:

**Core principle: Story > Tutorial, Insight > Instructions, Readable > Comprehensive**

1. **Title**: Intriguing but honest. "What I Learned Building X" > "How to Build X"
2. **Subtitle**: One sentence expanding on the title
3. **Opening**: Start with a story, observation, or surprising fact — not "In this article..."
4. **Body**: Mix narrative with technical content. Every section should answer "so what?"
5. **Closing**: Circle back to the opening story, end with a thought-provoking insight

### Writing Style

**Narrative-driven**
- Start with a concrete moment: "It was 2am and the deploy had failed again..."
- Weave technical details into a story arc
- Use transitions between sections, not just headings
- Each section should flow into the next

**Accessible depth**
- Explain complex concepts with analogies
- No unexplained jargon — if you use a term, make sure context makes it clear
- Code examples short and focused (< 15 lines)
- Link to resources for those who want to go deeper

**Visual rhythm**
- Short paragraphs (mobile reading)
- Pull quotes for key insights: `> The real bottleneck wasn't the algorithm — it was the data pipeline.`
- Bold for emphasis sparingly
- Images/diagrams to break up text
- Section breaks (`---`) between major ideas

**Avoid**
- "In this article, I will discuss..." (just start)
- Bullet-point-heavy content (that's a blog post, not an essay)
- Overly casual tone (Medium readers expect polish)
- Clickbait ("10 Things That Will Blow Your Mind")
- Academic dryness

### Medium-Specific Elements
- **Subtitle**: Appears below title, 1 sentence
- **Tags**: Up to 5, choose from popular Medium tags
- **Publication**: Target relevant publications (Towards Data Science, Better Programming, etc.)
- **Estimated read time**: Aim for 5-8 minutes (1200-2000 words)
- **Call to action**: "Follow for more" or "Clap if you found this useful"

### Structure Template

```

## [Second Major Point]

[Continue the story, go deeper]

---

[Circle back to opening, end with a memorable line]

```
