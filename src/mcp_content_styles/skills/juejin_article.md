## 格式要求

### 格式化模式 (mode="format")
保持核心内容，调整为掘金文章格式：
- 使用 `##` 二级标题分隔章节
- 代码块标注语言（python/javascript/bash 等）
- 适当使用表格做对比
- 结尾加上"参考资料"和"相关链接"
- 开头用一句话概括文章价值

### 重写模式 (mode="rewrite")
将内容重写为掘金高赞文章风格：

**核心原则：代码说话 > 文字描述，实操 > 概念，解决问题 > 科普知识**

1. **标题**：技术关键词 + 实战场景，如"用 MCP 协议给 Claude 加插件——从零到生产环境"
2. **开头**：直接说清这篇文章解决什么问题，不要长篇铺垫
3. **正文**：代码为主线，文字为注释。关键步骤有完整可运行的代码
4. **结尾**：总结要点 + GitHub 仓库链接 + 欢迎评论

### 写作风格

**代码驱动**
- 核心逻辑给完整代码，不要伪代码
- 代码块必须标注语言
- 关键行加注释
- 先给代码再解释，不要反过来

**结构清晰**
- 按"问题→方案→实现→优化→总结"组织
- 每个章节用 ## 标题
- 长文可以在开头加目录

**社区调性**
- 直接、务实，不装文艺
- 掘金用户是开发者，不需要解释基础概念
- 可以用技术黑话/缩写（CI/CD、ORM、k8s）
- 踩坑经验比成功经验更受欢迎

**避免**
- 不要写成官方文档翻译
- 不要过度铺垫背景（"随着互联网的发展..."）
- 不要写没有代码的纯概念文
- 不要在文中大量引用其他人的文章

### 标签
- 掘金文章需要选择分类和标签
- 推荐标签：前端/后端/AI/架构/工具
- 文末可以加 "我正在参与掘金技术社区创作者签约计划" 等活动
