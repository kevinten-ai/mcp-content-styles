## Format Requirements

### Format Mode (mode="format")
Adjust content to Bluesky post format:
- 300 character limit per post (grapheme count)
- Use thread format for longer content
- No hashtags (Bluesky doesn't use them for discovery)
- Links are auto-previewed with card embeds
- Alt text for any images

### Rewrite Mode (mode="rewrite")
Rewrite as Bluesky-native content:

**Core principle: Genuine > Viral, Thoughtful > Punchy, Community > Audience**

1. **Tone**: Authentic, slightly nerdy, no hustle-culture vibes
2. **Single post**: One clear thought, observation, or question
3. **Thread**: Genuine exploration of an idea, not a "content piece"
4. **Engagement**: Ask questions you actually want answers to

### Writing Style

**Bluesky culture**
- Anti-Twitter-hustle: No "thread incoming", no "like and repost"
- Genuine curiosity: "I've been thinking about..." "Has anyone tried..."
- Developer community: Technical discussions welcome, jargon OK
- Self-deprecating humor works well
- AT Protocol / decentralization topics get organic engagement

**Post format**
- 300 characters max (not 280 like X)
- No hashtags — discovery is via feeds and starter packs
- Links get rich card previews — don't waste characters describing them
- Images support alt text (always include it)
- Quote posts for commentary on others' posts

**Thread format**
- No numbering needed (1/N is not standard on Bluesky)
- Each post should stand on its own somewhat
- Keep threads short (3-6 posts)
- Don't pad — if you can say it in 2 posts, use 2

**Avoid**
- Engagement bait ("Hot take:", "Thread incoming")
- Hashtag stuffing (Bluesky doesn't use hashtags)
- Cross-posting verbatim from Twitter
- Self-promotion without substance
- "RT appreciated" / "Please share"

### Structure Templates


**Thread — Learning in public**
```
Post 1: [What you're exploring / working on today]

```

Please create Bluesky content following these guidelines. Write in English.
