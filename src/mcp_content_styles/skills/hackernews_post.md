## Format Requirements

### Format Mode (mode="format")
Adjust content to HN submission format:
- Title: factual, no clickbait, no emoji, no ALL CAPS
- If submitting a link: title should describe the content, not sell it
- If Show HN: "Show HN: [what it is] – [one line description]"
- If Ask HN: "Ask HN: [genuine question]"
- Comment text: plain text, no markdown formatting (HN doesn't support it)

### Rewrite Mode (mode="rewrite")
Rewrite as HN-appropriate content:

**Core principle: Substance > Style, Honesty > Hype, Technical > Popular**

1. **Title**: Factual and specific. "MCP Protocol: Standard Interface for AI Tool Integration" not "This Amazing Protocol Will Change AI Forever!"
2. **Show HN text**: What it does, why you built it, technical highlights, what you want feedback on
3. **Comments**: Add depth, share experience, be helpful
4. **Ask HN**: Genuine question with context on what you've already tried

### HN Culture

**What gets upvoted**
- Technical depth and original thinking
- Honest "Show HN" with real technical substance
- Sharing what you learned (not what you're selling)
- Thoughtful comments that add information

**What gets flagged/downvoted**
- Marketing disguised as a post
- Clickbait titles
- Self-promotion without substance
- "Launch" posts that read like press releases
- Emoji in titles
- Excessive enthusiasm ("Amazing!", "Game-changer!")

**Writing tone**
- Matter-of-fact, understated
- Let the work speak for itself
- Acknowledge limitations and tradeoffs
- "We built X because Y. It does A but not B. Feedback welcome."
- Technical precision matters

**Avoid**
- Any promotional language
- Buzzwords (disrupt, revolutionize, game-changing)
- Emoji anywhere
- Superlatives (best, fastest, most powerful)
- "We're excited to announce..."
- Asking for upvotes

### Submission Types

**Show HN**
```
Title: Show HN: [Name] – [factual one-line description]

Text:
[What it does — 1-2 sentences]

[Why you built it — the problem]

[Technical approach — what's interesting]

[Current status and limitations]

[What feedback you're looking for]

[Link to repo/demo]
```

**Ask HN**
```
Title: Ask HN: [Specific technical question]

Text:
[Context — what you're working on]

[What you've tried so far]

[Specific question]
```

**Link submission**
```
Title: [Descriptive, factual title about the content]
URL: [link]

(Optional first comment with context)
```

**Comment**
```
[Direct response to the point being discussed]

[Your experience or data that adds to the conversation]

[No pleasantries needed — get to the substance]
```

Please create Hacker News content following these guidelines. Write in English. No emoji. No marketing language.
