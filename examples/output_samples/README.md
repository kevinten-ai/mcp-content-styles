## 📖 原始内容

所有样例基于相同的原始 Markdown 内容：

```markdown
## MCP (Model Context Protocol) 介绍

MCP 是一种开放协议，用于标准化 AI 模型与外部工具的集成。

### 核心优势

- **标准化**: 统一的接口规范
- **互操作性**: 一次开发，到处运行
- **可扩展性**: 轻松添加新功能

### 应用场景

MCP 适用于各种 AI 增强型应用：
1. 代码编辑器（如 Cursor、Claude Code）
2. 数据分析工具
3. 自动化工作流
4. 智能客服系统

### 实际案例

以 Claude Code 为例，它通过 MCP 可以：
- 读取和修改本地文件
- 执行终端命令
- 与数据库交互
- 生成图像和视频

这让 AI 从"聊天机器人"变成了真正的"生产力工具"。

### 未来展望

随着 MCP 生态的发展，我们可以期待：
- 更多平台支持 MCP
- 更丰富的工具生态
- 更简单的开发体验
```

## 🎯 对比分析

### 同一内容的不同呈现

**知乎版本** - 专业深度：
- 采用"现象-原因-对策"结构
- 引用数据支撑观点
- 包含代码示例和技术细节
- 适合技术从业者深度阅读

**小红书版本** - 轻松种草：
- 使用"宝子们"等亲切称呼
- Emoji 丰富，视觉冲击强
- 痛点-解决方案-效果结构
- 适合快速浏览和分享

**抖音版本** - 视频脚本：
- 详细的镜头和画面标注
- 精确到秒的时间控制
- BGM 和转场效果提示
- 适合直接拍摄使用

**公众号版本** - 实战教程：
- 步骤清晰的实战指南
- 包含踩坑记录和优化建议
- 代码示例可直接复制使用
- 适合系统性学习

**微博版本** - 快速传播：
- 简短精炼，观点鲜明
- 话题标签增加曝光
- 引导评论互动
- 适合信息快速扩散

## 🔄 工作流程

```
原始 Markdown
    │
    ├──► 知乎 Prompt ──► LLM ──► 知乎文章 (~2000字)
    │
    ├──► 小红书 Prompt ──► LLM ──► 小红书笔记 (~300字)
    │
    ├──► 抖音 Prompt ──► LLM ──► 视频脚本 (~45秒)
    │
    ├──► 公众号 Prompt ──► LLM ──► 公众号文章 (~1500字)
    │
    └──► 微博 Prompt ──► LLM ──► 微博博文 (~200字)
```

## 📝 说明

这些样例展示了 MCP Content Styles 如何将相同的内容转化为不同平台的风格。实际使用时：

1. 准备你的原始 Markdown 内容
2. 选择目标平台和模式（format/rewrite）
3. 使用 `get_platform_prompt` 获取对应 Prompt
4. 将 Prompt 发送给 LLM 生成最终内容
5. 根据需要配图（可使用 mcp-image）
6. 发布到对应平台

---

**注意**: 以上样例仅供参考，实际输出会根据：
- 使用的 LLM 模型
- 模型参数（temperature 等）
- 原始内容质量
- 具体主题领域

等因素有所不同。
